# Правила WIP (нагрузки)

## 1. Назначение WIP

WIP используется **исключительно как аналитический показатель**.

WIP НЕ используется для:
- автоматической приоритизации
- блокировки задач
- принятия управленческих решений системой

## 2. Что считается активной нагрузкой

В активный WIP входят задачи со статусом:
- В работе

НЕ входят:
- Приостановленные
- Завершённые

## 3. Лимиты WIP

Лимиты:
- могут быть заданы
- но не являются жёсткими ограничениями

Превышение лимита:
- допускается
- обязательно визуализируется

## 4. Подсветка перегрузки

Перегрузка отображается:
- на уровне исполнителя
- на уровне проекта
- на уровне кармана

Цель подсветки:
- сигнализировать руководству
- дать основание для управленческого решения

## 5. Вытеснение задач

Любые рекомендации или предложения вытеснения системой запрещены.
Допустима только визуальная подсветка конфликтов.
