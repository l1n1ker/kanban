# UI Structure (Sprint 2)

## Цель документа
Описать структуру пользовательского интерфейса приложения Kanban IT:
- экраны системы
- элементы и действия пользователей
- навигацию и переходы

Документ опирается на:
`md_docs/architecture.md`, `md_docs/db_schema.md`, `md_docs/tasks_rules.md`,
`md_docs/wip_rules.md`, `md_docs/workflow.md`.
Новые сущности, статусы, роли или концепции не вводятся.

## Иерархия ролей

В системе установлена следующая иерархия ролей (по уровню прав):

admin > head > teamlead > curator > executor

Правило «или выше» означает:
роль, находящаяся левее в иерархии, обладает всеми правами ролей, находящихся правее.

Пример:
- если действие разрешено `curator`, его также могут выполнять `teamlead`, `head`, `admin`;
- если действие разрешено `teamlead`, его также могут выполнять `head`, `admin`.

`executor` не обладает правами вышестоящих ролей.

## Индикатор перегрузки отображается на уровнях:
- задача (подсветка карточки)
- проект (подсветка заголовка проекта)
- карман (подсветка заголовка кармана)

## Журнал действий отображается:
- на уровне задачи (детальная история)
- на уровне проекта и кармана — не отображается в текущей версии UI (данные хранятся системно)


## Основная панель навигации

**Назначение:**
Быстрый доступ к ключевым экранам и контекстным переходам.

**Доступные экраны из панели:**
- Карманы (список)
- Kanban-доска
- Timeline
- Пользователи (справочник)

**Дочерние экраны (через переходы из списков/карточек):**
- Проекты кармана (список)
- Проект (карточка и задачи)
- Задача (подробно)

## Экран: Карманы (список)

**Назначение:**  
Показать все карманы и их базовое состояние.

**Роли:**  
`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**  
- Таблица карманов
  - Поля: `id`, `name`, `status`, `date_start`, `date_end`,
    `owner_full_name`, `department`
- Фильтр по статусу кармана
- Кнопка/Действие: открыть карман

**Доступные действия:**  
- выбор фильтра по статусу  
  - результат: обновлённый список карманов
- переход в карман  
  - условие: выбран конкретный карман  
  - результат: открывается экран «Проекты кармана (список)»

**Переходы:**  
- к экрану «Проекты кармана (список)»

---

## Экран: Проекты кармана (список)

**Назначение:**  
Показать проекты выбранного кармана.

**Роли:**  
`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**  
- Метка/Подпись: выбранный карман (id, name, status)
- Таблица проектов
  - Поля: `id`, `name`, `status`, `date_start`, `date_end`,
    `curator_business_full_name`, `curator_it_full_name`
- Фильтр по статусу проекта
- Кнопка/Действие: открыть проект

**Доступные действия:**  
- выбор фильтра по статусу  
  - результат: обновлённый список проектов
- переход в проект  
  - условие: выбран конкретный проект  
  - результат: открывается экран «Проект (карточка и задачи)»

**Переходы:**  
- назад к экрану «Карманы (список)»  
- к экрану «Проект (карточка и задачи)»

## Экран: Проект (карточка и задачи)

**Назначение:**  
Показать карточку проекта и список задач проекта.

**Роли:**  
`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**  
- Панель/Карточка проекта
  - Поля: `id`, `name`, `status`, `date_start`, `date_end`,
    `curator_business_full_name`, `curator_it_full_name`, `pocket_id`
- Таблица задач проекта
  - Поля: `id`, `description`, `status`, `date_created`,
    `date_start_work`, `date_done`, `executor_full_name`, `customer`, `code_link`
- Фильтр по статусу задачи
- Фильтр по исполнителю
- Кнопка/Действие: создать задачу
- Кнопка/Действие: открыть задачу
- Кнопка/Действие: перейти на Kanban-доску (по текущему проекту)
- Кнопка/Действие: перейти на Timeline (по текущему проекту)

**Доступные действия:**  
- создание задачи (`admin`/`head`/`teamlead`/`curator`)  
  - входные данные: описание, заказчик, исполнитель (выбор пользователя), ссылка на код  
  - результат: задача создаётся со статусом `Создана`, сохраняется `executor_user_id`
- фильтрация задач  
  - результат: обновлённый список задач
- переход к задаче  
  - условие: выбран конкретный id  
  - результат: открывается экран «Задача (подробно)»
- переход на Kanban/Timeline  
  - условие: выбран проект  
  - результат: открывается соответствующий экран с фильтром по проекту

**Переходы:**  
- назад к экрану «Проекты кармана (список)»  
- к экрану «Задача (подробно)»  
- к экрану «Kanban-доска»  
- к экрану «Timeline»

## Экран: Задача (подробно)

**Назначение:**  
Показать подробную информацию о задаче и её приостановках.

**Роли:**  
`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**  
- Панель/Карточка задачи
  - Поля: `id`, `description`, `status`, `date_created`,
    `date_start_work`, `date_done`, `executor_full_name`, `customer`,
    `code_link`, `project_id`
- Таблица приостановок (task_pauses)
  - Поля: `id`, `date_start`, `date_end`
- Панель/Вкладка: История
  - Список записей журнала по задаче в хронологическом порядке
  - Формат записи: `DD.MM.YYYY HH:MM — ФИО пользователя — Описание действия (Комментарий)`
- Кнопка/Действие: начать работу
- Кнопка/Действие: приостановить
- Кнопка/Действие: возобновить
- Кнопка/Действие: завершить

**Доступные действия:**  

Изменение статуса задачи возможно только в соответствии
с допустимыми переходами из `architecture.md`
и с учётом ролей:

- Начать работу  
  - роли: `curator` и выше  
  - результат: статус → `В работе`

- Приостановить  
  - роли: `curator` и выше  
  - результат: статус → `Приостановлена`

- Возобновить  
  - роли: `curator` и выше  
  - результат: статус → `В работе`

- Завершить  
  - роли: `curator` и выше  
  - результат: статус → `Завершена`

`executor` имеет доступ только на чтение
и не может изменять статус задачи.

Добавление приостановки:
- роли: `curator` и выше  
- входные данные: `date_start`, `date_end`  
- результат: запись в `task_pauses`


**Переходы:**  
- назад к экрану «Проект (карточка и задачи)»

## Экран: Kanban-доска

**Назначение:**  
Показать задачи по статусам для выбранного кармана/проекта.

**Роли:**  
`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**  
- Панель фильтров
  - Фильтр: карман
  - Фильтр: проект
  - Фильтр: исполнитель
  - Фильтр: статус задачи
- Панель/Список колонок по статусам:
  - `Создана`
  - `В работе`
  - `Приостановлена`
  - `Завершена`
- Карточки задач (в колонках)
  - Поля: `id`, `description`, `executor_full_name`,
    `date_start_work`, `date_done`, `code_link`
- Метка/Подпись: индикатор перегрузки (визуальная подсветка)

# Визуализация перегрузки:
- уровень задачи — подсветка карточки задачи;
- уровень проекта — подсветка заголовка проекта (если выбран фильтр по проекту);
- уровень кармана — подсветка выбранного кармана в панели фильтров.

**Доступные действия:**  
- выбор фильтров  
  - результат: обновление задач на доске
- изменение статуса задачи (если разрешено ролью)  
  - результат: задача перемещается в колонку по допустимому переходу
- открытие задачи  
  - результат: переход на экран «Задача (подробно)»

**Переходы:**  
- к экрану «Задача (подробно)»  
- к экрану «Timeline»  
- к экрану «Проект (карточка и задачи)» (если выбран проект)

`admin`, `head`, `teamlead`, `curator` (полный доступ),  
`executor` (только чтение).

**Элементы:**

  - Фильтр: исполнитель
  - Фильтр: статус задачи

    `executor_full_name`
- Метка/Подпись: приостановки отображены как отдельные интервалы
- Метка/Подпись: индикатор перегрузки (визуальная подсветка)

- уровень задачи — подсветка карточки задачи;
- уровень проекта — подсветка заголовка проекта (если выбран фильтр по проекту);

## Экран: Пользователи (справочник)

**Назначение:**  
Просмотр, создание и редактирование пользователей системы.

**Роли:**  
`admin` — полный доступ,  
`head` — просмотр,  
остальные роли — нет доступа.

**Элементы:**  
- Таблица пользователей
  - Поля: `full_name`, `login`, `role`, `is_active`
- Кнопка/Действие: добавить пользователя
- Кнопка/Действие: редактировать пользователя
- Кнопка/Действие: активировать/деактивировать

**Доступные действия:**  
- создание пользователя вручную  
  - входные данные: ФИО, Windows-логин, роль, активность  
  - результат: запись в `users`
- редактирование ФИО и роли  
  - входные данные: ФИО, роль  
  - результат: обновление `users.full_name`, `users.role`
- отключение пользователя без удаления  
  - входные данные: флаг активности  
  - результат: обновление `users.is_active`

**Переходы:**  
- к экрану «Карманы (список)»  
- к экрану «Kanban-доска»  
- к экрану «Timeline»

---

## Использование пользователей в других экранах

- Во всех формах выбора исполнителя, куратора, владельца используется поле
  типа «Список» с отображением `users.full_name`.
- В результатах сохраняется `user_id`, а не логин.
