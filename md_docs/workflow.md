# Общий процесс работы (Workflow)

## 1. Структура

Работа организована по иерархии:

Карман  
→ Проект  
→ Задача  

## 2. Карман

Карман:
- логическая группа проектов
- имеет только два состояния:
  - Запущен
  - Завершён

Карман не используется для приоритизации.

## 3. Проект

Проект:
- всегда стартует
- не может быть отменён системой
- может содержать любое количество задач

Проект может быть завершён только куратором (curator) и выше.

## 4. Задачи

Задачи создаются:
- только внутри проекта
- создавать задачи может curator и выше

Исполнение задач:
- параллельное
- с частой сменой контекста
- с возможной приостановкой

## 5. Роль системы

Система:
- фиксирует факты
- считает сроки
- визуализирует нагрузку
- подсвечивает проблемы

Система НЕ:
- управляет приоритетами
- оптимизирует план
- принимает решения за человека

## 6. Временная визуализация

Для анализа используется:
- Kanban
- базовый Timeline

Особенности:
- приостановки отображаются явно
- приостановки вычитаются из сроков
- перегрузка видна во времени

---

## 7. Нормативное уточнение Sprint 2 (приоритет над предыдущими разделами)

При конфликте с предыдущими разделами этого документа применяется данный раздел.

- Процессная модель «Карман → Проект → Задача» сохраняется без изменений.
- Статусы сущностей в Sprint 2 ведутся через единый справочник `statuses`.
- Для пользователей активность/доступность отражается статусом пользователя (`users.status_id`), а не отдельным булевым флагом.
- Изменение модели хранения статусов не меняет управленческие правила процесса.
