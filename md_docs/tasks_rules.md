# Правила работы с задачами

## 1. Общие положения

Задача — минимальная единица управляемой работы ИТ-подразделения.

Все задачи:
- принадлежат проекту
- косвенно принадлежат карману через проект
- создаются и управляются **кураторами проектов**
- на старте имеют одного исполнителя

## 2. Роли и ответственность

### Кураторы проекта (бизнес / ИТ) - curator
- создают задачи
- изменяют описание и связи
- приостанавливают и возобновляют задачи
- завершают задачи

### Тимлид / начальник отдела / админ - head/teamlead/admin
- имеют все права кураторов
- могут игнорировать лимиты WIP
- используют систему для выявления перегрузки

### Исполнитель (ИТ)
- не управляет состояниями задачи
- является объектом учёта нагрузки

## 3. Жизненный цикл задачи

Возможные состояния:
- Создана
- В работе
- Приостановлена
- Завершена

создавать задачи может curator и выше

### Приостановка задачи
- фиксируется явно
- имеет дату начала и окончания
- исключается из:
  - активного WIP
  - расчёта фактического срока выполнения
- приостанавливать и возобновлять задачи могут curator и выше.

### Сроки
- сроки считаются в днях
- приостановки вычитаются из фактического срока

## 4. Конфликты между проектами

Если один исполнитель задействован в нескольких проектах:
- система **не решает конфликт**
- решение принимают кураторы проектов
- одна задача может быть приостановлена в пользу другой

Система лишь:
- фиксирует факт
- подсвечивает перегрузку
